# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | `@expo/vector-icons/Ionicons` |
| Font | `Instrument Sans` locally bundled; fallback `sans-serif` |

### Decision Sources

- `CONTEXT.md`: tabs from day one, `Home` / `History` / `Settings`, dedicated first-run privacy screen, Apple Health-inspired visual direction, polished empty states, no "coming soon" treatment.
- `RESEARCH.md`: Expo Router `Stack` + stable JavaScript `Tabs`, offline-safe launch, one-time local privacy acknowledgement, short splash, no network bootstrap.
- `REQUIREMENTS.md`: no sign-in, local-only explanation on first run, app shell usable offline.
- Defaults used here: spacing tokens, typography scale, exact color values, exact copy, component naming.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Hairline gaps, icon-label separation inside pills |
| sm | 8px | Tight content grouping inside cards |
| md | 16px | Default card padding and vertical rhythm |
| lg | 24px | Section padding and inter-card spacing |
| xl | 32px | Screen horizontal padding on major sections |
| 2xl | 48px | Break between hero content and supporting sections |
| 3xl | 64px | Top-of-screen breathing room below status/header area |

Exceptions: interactive hit areas may expand to a minimum `44x44` without changing visual spacing; bottom tab bar uses `56px` visual height with `16px` top padding from screen content.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.35 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

Rules:

- Use exactly two weights: `400` and `600`.
- Body copy defaults to `16px/1.5`; do not use `14px` for paragraph text.
- Section titles use `20px/600`; the privacy screen headline and dashboard top title may use `28px/600`.
- Tab labels and compact metadata use `14px/600`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F5F6F2` | App background, privacy screen background, large empty surfaces |
| Secondary (30%) | `#FFFFFF` | Cards, tab bar surface, inset panels, rows |
| Accent (10%) | `#4A7CFF` | Primary CTA, active tab icon/label, focused control outline, emphasized inline link text |
| Destructive | `#D65252` | Destructive actions only |

Accent reserved for: privacy-screen primary button, active bottom-tab state, tappable inline privacy link in Settings, keyboard/focus outline on actionable rows. Do not use accent for passive card chrome, decorative icons, or empty-state illustrations.

Supporting neutrals:

- Primary text: `#111827`
- Secondary text: `#6B7280`
- Divider/hairline: `#E5E7EB`
- Soft tint for placeholder panels: `#EEF2F7`

Future medical interpretation colors are out of scope for this phase and must not be introduced yet.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Continue Offline |
| Empty state heading | No readings yet |
| Empty state body | When you record a blood pressure reading, it will appear here and stay on this device. |
| Error state | Something went wrong loading your local app shell. Close and reopen the app. If the problem continues, restart the device and try again. |
| Destructive confirmation | None in Phase 1. No destructive actions ship in this phase. |

Additional locked copy:

- Privacy title: `Private by default`
- Privacy body: `Your blood pressure readings stay on this device. No account, cloud sync, or ads are required to use the app.`
- Privacy support line: `This app helps you log and review readings. It does not provide medical diagnosis or treatment advice.`
- History empty title: `No history yet`
- History empty body: `Saved readings will appear here in reverse chronological order.`
- Settings privacy row label: `Local storage only`
- Settings privacy row detail: `Your data stays on this device unless you choose to export a PDF later.`

Tone rules:

- Calm, direct, and non-clinical.
- No startup language such as `Get started`, `Welcome`, or `Coming soon`.
- No diagnostic or advisory phrasing.

## Shell Behavior

### Launch Flow

1. App opens to splash.
2. Splash remains only long enough to load local fonts, theme constants, and the one-time privacy acknowledgement flag.
3. If privacy acknowledgement is missing, route to the dedicated privacy screen.
4. After acknowledgement, route into the tab shell and do not show the privacy gate again on normal launch.

### Navigation Contract

- Use Expo Router root `Stack` with a dedicated `privacy` route plus a `(tabs)` group.
- Use stable JavaScript `Tabs`, not native alpha tabs.
- Tabs are fixed in this order: `Home`, `History`, `Settings`.
- All three tabs are visible in Phase 1 and styled as durable product surfaces.
- No badges, no modal tab actions, no floating action button in Phase 1.
- The active tab tint uses the accent color; inactive tabs use secondary text color.

### Offline Contract

- Phase 1 renders fully without network access.
- No auth, analytics, remote config, OTA check, or remote font fetch may block first paint.
- If a network is unavailable, there is no banner or warning because offline is the intended default mode.

### Header And Screen Chrome

- Each tab screen uses a large top title aligned left with generous top breathing room.
- Avoid dense nav bars. The shell should feel open, with content leading rather than chrome.
- Cards use `16px` corner radius, white surfaces, and subtle `#111827` at `6%` shadow/overlay depth only.
- Dividers are rare; prefer spacing over borders.

## Empty-State Structure

### Home

Home must feel like a finished health dashboard even with zero readings. Structure it in this order:

1. Large title: `Blood Pressure`
2. Summary hero card with placeholder reading slots and the heading `No readings yet`
3. Secondary card explaining local-only behavior in one short paragraph
4. Trend preview card with muted placeholder line treatment and copy indicating trends appear after saved readings

Rules:

- Do not center the whole screen like a blank slate.
- Do not use illustration-first empty states.
- The hero card is the visual anchor and should occupy the strongest position above the fold.

### History

History must resemble the future final list screen. Structure it in this order:

1. Large title: `History`
2. Small supporting sentence about saved readings appearing in reverse chronological order
3. List container card with empty rows omitted and one clear empty-state message inside the list area

Rules:

- Keep the list container visible even when empty.
- No fake sample data.
- No `Coming soon` or backlog language.

### Settings

Settings is informational in Phase 1 and establishes the future product frame. Structure it in this order:

1. Large title: `Settings`
2. Privacy section with rows for `Local storage only` and `About this app`
3. Secondary muted footnote restating that export is optional and happens only when the user chooses it in a later phase

Rules:

- Rows may be static or non-navigating if the destination is not built yet, but they must still look like real settings rows.
- Do not present unavailable rows as disabled controls unless interaction truly exists.

## Component/System Decisions

Core primitives to establish in Phase 1:

- `ScreenContainer`: shared safe-area wrapper with top breathing room and horizontal padding.
- `ScreenTitle`: large left-aligned title component using `Display` or `Heading` role.
- `SurfaceCard`: white card with `16px` radius, `16px` or `24px` padding, minimal shadow.
- `InfoRow`: settings-style row with title, optional detail, and optional chevron.
- `EmptyStatePanel`: reusable empty-state block for History and supporting Home sections.
- `PrimaryButton`: solid accent button reserved for the privacy-screen acknowledgement.
- `TabIcon`: Ionicons outline style by default; filled variant only for active state where available.

System rules:

- Radius scale: `12px` for small inset panels, `16px` for standard cards, `24px` only for the privacy hero card.
- Shadow scale: one shadow style only in Phase 1.
- Motion: optional fade/slide-in on privacy card and home cards; duration `180-240ms`, ease-out, no spring bounce.
- Icons stay outline/lightweight; avoid filled icon sets except for active tab emphasis.
